The metadata model

The metadata model of the FOBL real estate catalogue is based on the Swiss DCAT Application Profile (DCAT-AP CH v3.0.0) and extends it minimally with specific components for managing the Confederation's real estate data.

Core architecture

The model consists of five main DCAT classes and one FOBL-specific extension:

DCAT standard classes:

  • dcat:Catalog - FOBL real estate catalogue as overall directory
  • dcat:Dataset - Real estate data collections (building master data, energy data)
  • dcat:DatasetSeries - Time series (annual reports, cost developments)
  • dcat:Distribution - Distribution forms (Excel, API, PDF)
  • dcat:DataService - Data services (SAP interfaces, REST APIs)

Schema modelling (DCAT-compliant):

  • dct:Standard - Business object schema (building, land parcel, lease)
  • skos:ConceptScheme - Controlled vocabularies and value lists (building types, cantons)
  • skos:Concept - Individual values in value lists

FOBL-specific extension:

  • bv:Attribute - Data fields with type definitions, validation rules and value list references

Business object modelling

dct:Standard "Gebäude-Schema"
├── dct:identifier: "STD-001"
├── dct:title: "Gebäude"@de, "Bâtiment"@fr, "Edificio"@it
├── dct:description: Definition gemäss KBOB/TERMDAT
├── bv:hasAttribute:
│   ├── bv:Attribute "Baujahr"
│   │   ├── dct:type: "xsd:gYear"
│   │   ├── dct:required: true
│   │   └── dct:conformsTo → skos:ConceptScheme "Jahre"
│   └── bv:Attribute "Gebäudetyp"
│       └── dct:conformsTo → skos:ConceptScheme "Gebäudetypen"
└── dcat:version: "3.0.0"

Versioning

All elements support full version control:

  • dcat:version - Semantic versioning (MAJOR.MINOR.PATCH)
  • dct:issued - Initial publication
  • dct:modified - Last modification
  • adms:versionNotes - Change description
  • dcat:previousVersion - Reference to previous version

Governance and access control

Role model:

  • bv:dataOwner (foaf:Agent) - Business manager for real estate management
  • bv:dataSteward (foaf:Agent) - Data architect / data manager
  • bv:dataCustodian (foaf:Agent) - Technical system administrator

Access rights and compliance:

  • dct:accessRights - Access restrictions (public/internal/confidential)
  • dct:license - Terms of use
  • bv:dsgCategory - FADP classification
  • bv:isgClassification - Information security level
  • dcatap:applicableLegislation - Legal bases (OIAF, FADP)

Structure example

BBL Immobilienkatalog (dcat:Catalog)
├── Geschäftsobjekt-Schemas (Konzeptuelle Ebene)
│   ├── dct:Standard "Gebäude-Schema" v3.0.0
│   └── dct:Standard "Grundstück-Schema" v2.1.0
│
├── Datenattribute (Logische Ebene)
│   ├── bv:Attribute "EGID" (required, unique)
│   └── bv:Attribute "Nutzfläche" (decimal, m²)
│
├── Wertelisten (Referenzdaten)
│   ├── skos:ConceptScheme "Gebäudetypen" v2.1.0
│   └── skos:ConceptScheme "Kantone" v1.0.0
│
└── Datasets (Physische Ebene)
    ├── dcat:Dataset "Gebäudestammdaten 2025"
    │   └── dcat:Distribution (SAP-Export, REST-API)
    └── dcat:DataService "IMMO-API v2"

This structure enables full traceability of data provenance, clearly defined responsibilities and meets all regulatory requirements of the Federal Administration.

Further information and standards

Standards and specifications used:

  • DCAT v3 (W3C Recommendation)
    The foundation of our metadata model. The Qualified Forms for extended relationships are particularly relevant:
    https://www.w3.org/TR/vocab-dcat-3/#qualified-forms
  • Dublin Core Metadata Terms
    Defines core properties such as dct:Standard, dct:conformsTo and dct:identifier:
    https://www.dublincore.org/specifications/dublin-core/dcmi-terms/#Standard
  • DCAT-AP CH (eCH-0200)
    The Swiss application standard for data portals of the Federal Administration:
    https://ech.ch/de/ech/ech-0200/3.0
    https://www.dcat-ap.ch/
  • SKOS (Simple Knowledge Organization System)
    For modelling value lists and controlled vocabularies:
    https://www.w3.org/TR/skos-reference/

Governance roles according to NaDB

The role definitions are based on the National Data Management of the FSO:

  • Data Owner - Business responsibility
  • Data Steward - Data architecture and quality
  • Data Custodian - Technical operations
